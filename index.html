Termer och begrepp

Element

Element behövs när man skapar en html sida. De har alltid en starttagg och oftast även en sluttagg, och däremellan ett innehåll.

HTML

HTML är en förkortning av Hyper Text Markup Language och är det språk som används när man skapar sidor på nätet. Det är inte ett programmeringsspråk, utan ett märkspråk. Ett programmeringsspråk utför kommandon, men ett märkspråk används för att märka upp syftet med olika texter. Vad som är rubrik, underrubrik eller brödtext till exempel.

Responsive

En websida som är responsive förändrar sig (responderar/svarar) till den skärm du använder. Så layouten beror på den funktion din webläsare har, om du tittar från till exempel en dator, ipad eller en android. En sida som inte är responsive däremot har alltid samma layout vilket får till följd att den kan fungera dåligt och vara svårläst om du sitter vid en annan skärmtyp än vad sidan utvecklats för.

ID

Ett ID är ett attribut som används för att identifiera ett element, och det måste vara unikt. Man kan använda ID för att skilja huvudmeny från sidomenyer, eller för att skapa ett "ankare" som när man klickar på det tar dig till det elementet.

Class

Class är ett attribut, som till skillnad mot ID som är unikt eftersom identifierar ett element istället grupperar (klassificerar) element. Det innebär att det kan finnas flera element av samma Class på samma sida. Det kan vara en en klass som anger att texten ska bli rödmarkerad, och det kan natuligtvis finnas rödmarkerad text på flera ställen i texten.

Meta

Meta är ett prefix som betyder gömd. Meta är delen som inte är synligt, men som påverkar egenskaperna. De stora sökmotorerna använder sig av sökorden som finns här för att hitta just den sidan. En annan typ av Meta är headerinformation som inte visas, men används för kommunikation mellan webläsaren och webservern redan innan htmlkoden läses in.

http

http står för Hyper Text Transfer Protocol och är ett kommunikationsprotokoll för som används för att ladda websidor på internet via www (World Wide Web). Hypertext är en text som har så kallade hyperlänkar som kopplar ihop en text med en text på en annan sida. Ett vanligt fenomen på till exempel Wikipedia, där ett ord kan ha en länkning som tar dig till en annan sida som förklarar det ordet. Ordet Transfer syftar på att det handlar om att skicka och ta emot information. För att kommunikationen ska fungera krävs regler, och Protocol syftar just på det sätt som man kommit överrens att kommunicera. Http används av webläsaren när den ska hämta information från en webserver. Det som händer är att webläsaren (klienten) skickar någon form av förfrågning till webservern.

    #GET - klienten ber servern hämta en fil
    #HEAD - klienten ber servern om information om innehåll utan att skicka över det
    #POST- klienten ber servern att lagra viss information
    #PUT - klienten laddar upp en fil på servern
    #DELETE - klienten ber servern ta bort en fil
    #TRACE - klienten ber servern skicka tillbaks en förfrågan, används för att kontrollera om tredje part har manipulerat data
    #OPTIONS - visar en lista över samtliga http kommandon
    #CONNECT - används för att omvandla frågan till en TCP/IP tunnel, tex för att möjliggöra SSL/HTTPS
    #PATCH - sätt att delvis byta ut innehållet i en fil på servern

https

Idag är det fler och fler sidor som använder https istället för http, vilket beror på att https är säkrare. Att det är säkrare beror på att man använder sig av kryptering av överföring via SSL och det därmed är svårare för tredje part att komma åt den. Dessutom rankar Google sidor som använder https högre, vilket ytterligare motiverar att man ska använda sig av https på sin webplats. Sidor som använder sig av https har ett hänglås i adressfältet i webbläsaren.

compressed data

Compressed data kan man använda sig av för att spara lagringsutrymme eller för snabbare överföring av information. Detta eftersom det innebär att datan komprimeras på så sätt att färre informationsbärande enheter (oftast bitar) behövs

CDN

CDN står för Content Delivery Network och är en central del i hur ljud, bild och liknande tjänster levreras till användare på internet. Många tjänster på internet är bandbreddsintensiva både gällande tid och volym. CDN-tekniken gör att belastningen sprids ut över flera servrar så att tjänsten kan leveras med hög prestanda och tillgänglighet. Att använda sig av CDN är det snabbaste sättet att komma igång med Boostrap.

local bootstrap

Ett alternativ till att använda sig av CDN är att ladda ner en kopia av Bootstrap lokalt till ditt projekt.

JQuery

JQuery är världens mest använda Java Script Biblilotek, utformat för att göra det enklare att html, css och DOM modifiera element på hemsidan och hantera händelser. Det stödjer också plugins, varav många är gratis att ladda ner och använda.

Flexbox

Flexbox är ett sätt att skapa layouten för webbsidor. Flexboxen förenklar att få raka kolumner och liknande problem vid skapandet av sidan. Några begrepp att hålla reda på för att förstå Flexbox.

    #Flexbehållare - Föräldraelement vari allt som ska hanteras av Flexboxen placeras
    #Flexelement - de underelement som placeras i Flexbehållaren
    #Flexaxel - anger i vilken riktning elementen ska placeras, antingen rad eller kolumn. Från vänster till höger
    #Omvända fleraxlar - Används när man istället vill gå från höger till vänster
    #Korsaxel - En axel som är förskjuten 90 grader från flexaxeln, dvs är fleraxeln horisontell är korsaxeln vågrät

Grid

Ett tänkt rutnät på en hemsida som används för att styra vilkor och regler när man konstuerar en websid kallas Grid. CSS Grid är ett alternativ till Flexbox, men kräver mer kod än Flexbox. CSS Grid är bättre att använda sig av när man vill skapa tvådimensionella element, dvs som är både en rad och en kolumn.

Code snippets

Code snippets är små bitar av återanvändbar kod som man kan använda sig av när man skapar en sida genom att anropa blocket på ett eller annat sätt
